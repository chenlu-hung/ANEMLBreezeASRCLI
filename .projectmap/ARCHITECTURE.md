## Overview
`breeze-asr` is a headless macOS CLI that transcribes audio/video to SRT subtitles using the
on-device Breeze-ASR-25 model (via WhisperKit), and can synthesise a time-aligned dub from a
translated SRT. It is ASR-only — subtitle correction/translation are deliberately left to an
external LLM step. The architecture is a thin arg-parsing entry point delegating to focused
services: a transcription pipeline (FFmpeg → WhisperKit → SwiftSubtitles) and a dubbing pipeline
that by default synthesises per-cue speech via the cloud edge-tts engine (native SwiftEdgeTTS,
fixed neural voice) — with local indextts2 voice cloning as an opt-in (`--clone`/`--ref`) — then
does timeline placement (default stretch-video) → FFmpeg mux.

## Entry points
- `Sources/breeze-asr/BreezeASRCLI.swift` — `@main`; `breeze-asr <input>` (transcribe) and `breeze-asr dub <video> --srt …` (dub)

## Build / test
- **build**: `swift build` (release: `swift build -c release` → `.build/release/breeze-asr`)
- **test**: `swift test` (no test target defined yet)
- Requires FFmpeg on PATH (`brew install ffmpeg`); default `dub` TTS is cloud edge-tts (needs internet,

## Conventions
- **stdout = result, stderr = progress/logs.** Transcribe prints the SRT path; dub prints the
  output video path. Everything else (progress bars, timeline reports) goes to stderr so callers
  can capture the result cleanly.
- **Sync `main` + `dispatchMain()`**: work runs on a detached Task while the main thread services
  the main queue. Loading the CoreML model on the main thread of an `async` main deadlocks.
- **Compute units default to `.cpuAndNeuralEngine` (ANE); `--compute gpu` switches to `.cpuAndGPU`.**
  ANE wins on Apple silicon (NPU > GPU for this model). ANE is *not* a deadlock in this unsigned CLI
  — the first ANE run does a one-time ~40 min cold compile of the 1.2 GB encoder, then
  ANECompilerService caches it per-bundle-id (`breeze-asr`) and later loads take ~3 s. VibeTyping's
  warm ANE cache can't be inherited (e5rt cache is keyed per bundle id; only `.mlmodelc` files are
  shared). On a cold cache the first default run looks like a hang — use `--compute gpu` to skip it.
- **Model cache reuse** before download: WhisperKit looks in app-support HubCaches and validates
  a real (>100 MB) AudioEncoder before trusting a cached model; only downloads ~3 GB if none found.
- **`tokenizerFolder` is pinned** to an app-support cache to avoid TCC-protected `~/Documents/huggingface`.
- Errors are typed `LocalizedError` enums per service; `CLIError` carries user-facing messages.
